# Автотор · Kandidat (тестирование кандидатов)

Веб-приложение для проведения тестов по направлениям (программирование, embedded, данные, ИИ), панели рекрутера и опциональной **оценки резюме через локальную LLM (Ollama)**.

## Стек

- **Frontend:** React, Vite, Tailwind CSS, Wouter, TanStack Query  
- **Backend:** Express (TypeScript), единый сервер отдаёт API и клиент в dev/production  
- **Данные сессий и ответов:** JSON-файл `server/data/db.json` (в репозиторий не коммитится)  
- **Вопросы:** `server/data/questions.json`  
- **pnpm** — менеджер пакетов

## Требования

- **Node.js** 20+ (рекомендуется LTS)  
- **pnpm** 9+ (`corepack enable` или `npm i -g pnpm`)  
- **Ollama** — только если нужна кнопка оценки резюме в панели рекрутера (`/dashboard`)

## Установка и запуск

```bash
cd quiz
pnpm install
pnpm dev
```

По умолчанию сервер слушает **порт 5000** (см. лог при старте). Откройте в браузере указанный адрес.

Сборка и продакшен:

```bash
pnpm build
pnpm start
```

## Переменные окружения

| Переменная       | Назначение |
|------------------|------------|
| `NODE_ENV`       | `development` / `production` (ставится скриптами) |
| `PORT`           | Порт HTTP-сервера (по умолчанию **5000**) |
| `DATABASE_URL`   | Нужна **только** для миграций Drizzle (`pnpm db:push`), если используете PostgreSQL из схемы. Обычный запуск тестов и дашборда на JSON **не требует** базы. |

## Ollama: установка и запуск

Ollama — локальный сервер LLM. Проект ходит к нему **с бэкенда** (прокси `POST /api/ollama/generate`), чтобы обойти CORS и не светить API в браузере.

### 1. Установка

- **Windows / macOS / Linux:** [https://ollama.com/download](https://ollama.com/download)  
- После установки в терминале доступна команда `ollama`.

### 2. Запуск службы

Обычно Ollama ставится как сервис и сам слушает порт **11434**.

Проверка:

```bash
curl http://localhost:11434/api/tags
```

Если ответ JSON со списком моделей — сервис запущен.

Запуск вручную (если сервис не поднят):

```bash
ollama serve
```

Оставьте процесс работать, пока используете оценку резюме.

### 3. Модель по умолчанию

В коде заданы:

- **URL Ollama:** `http://localhost:11434/api/generate` (файл `server/routes.ts`)  
- **Имя модели по умолчанию:** `gemma3:4b` (и на сервере, и в `client/src/pages/dashboard.tsx`, константа `LLM_MODEL`)

Скачайте модель один раз:

```bash
ollama pull gemma3:4b
```

Другая модель — после `ollama pull <имя>` замените строку модели:

- в **`server/routes.ts`** — значение `model: body.model ?? "gemma3:4b"`  
- в **`client/src/pages/dashboard.tsx`** — `const LLM_MODEL = "..."`

Имя должно совпадать с тем, что показывает `ollama list`.

### 4. Как подключается приложение

1. Кандидат проходит тест на главной странице.  
2. Рекрутер в **`/dashboard`** загружает резюме к сессии (при необходимости).  
3. Запрос оценки идёт на **`POST /api/ollama/generate`** с телом JSON (упрощённо): `model`, `prompt` (можно пустым — подставится промт из `server/prompts/resume-evaluation.txt`), `stream: true`, опционально `sessionId` — тогда к промту добавляется текст резюме.  
4. Сервер проксирует поток ответа Ollama в формате NDJSON в UI.

Если Ollama не запущена или модель не скачана, в интерфейсе будет ошибка соединения (статус 502 / сообщение от прокси).

### 5. Ollama на другом хосте или порту

Сейчас адрес **жёстко** задан как `http://localhost:11434`. При необходимости вынесите базовый URL в переменную окружения (например `OLLAMA_HOST`) и подставьте её в `fetch(...)` в `server/routes.ts` — после изменения перезапустите `pnpm dev`.

## Структура данных (локально)

- `server/data/questions.json` — вопросы по ролям  
- `server/data/db.json` — сессии и ответы (создаётся при работе; не коммитится)  
- `server/data/resumes/` — загруженные файлы резюме (не коммитится)  
- `server/prompts/resume-evaluation.txt` — текст промта для LLM по умолчанию
